# Спецификация HTTP API 

Эта документация описывает работу HTTP REST API сервера `anet-auth` (актуально для версии протокола `ASTP v0.6+`).

Все запросы по умолчанию имеют базовый префикс `/api/v1`.
Интерфейс делится на три зоны: **публичную** (для скачивания конфигов клиентами), **административную** (для WebUI/управления, защищена JWT) и **инфраструктурную** (для связи VPN-нод с бэкендом, защищена токеном ноды).

---

## 1. Авторизация Администратора (Admin Auth)

Используется для получения токена доступа к панели управления. Время жизни сессии в БД — 12 часов.

### `POST /login` — Вход в панель управления
* **Тип запроса**: Публичный
* **Тело запроса (JSON)**:
  ```json
  {
    "login": "admin",
    "password": "your_secure_password"
  }
  ```
* **Ответ (200 OK)**:
  ```json
  {
    "access_token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9..."
  }
  ```
* **Ошибки**:
    * `401 Unauthorized`: Неверный логин или пароль.

## 2. Управление локациями (Nodes/Servers)

Все запросы в этой секции требуют заголовка авторизации: `Authorization: Bearer <access_token>`.

### `GET /servers` — Получить список всех нод
* **Ответ (200 OK)**:
  ```json
  [
    {
      "id": "5ca3580d-8488-457b-85e6-7013239747c4",
      "name": "Germany VPS 1",
      "address": "127.0.0.1",
      "public_key": "ecfNEF2kYJ8kjyUqlI8HgBZaAnPBmsZVequuoxjZppk=",
      "quic_port": 4519,
      "ssh_port": 822,
      "vnc_port": 56678,
      "ssh_user": "hanyuu"
    }
  ]
  ```

### `POST /servers` — Добавить физический сервер (ноду)
* **Тело запроса (JSON)**:
  ```json
  {
    "name": "Germany VPS 1",
    "address": "127.0.0.1",
    "public_key": "ecfNEF2kYJ8kjyUqlI8HgBZaAnPBmsZVequuoxjZppk=",
    "quic_port": 4519,
    "ssh_port": 822,
    "vnc_port": 56678,
    "ssh_user": "hanyuu"
  }
  ```
* **Ответ (200 OK)**: Возвращает созданный объект сервера с присвоенным UUID.

## 3. Управление клиентами (VPN Users)

Все запросы требуют заголовка: `Authorization: Bearer <access_token>`.

### `GET /users` — Список клиентов (с пагинацией)
* **Параметры запроса (Query)**:
    * `from`: Смещение от начала (по умолчанию `0`)
    * `limit`: Количество записей (по умолчанию `50`)
* **Ответ (200 OK)**:
  ```json
  {
    "items": [
      {
        "id": "aabc9d26-fdf8-46d4-8245-c2d683b9f075",
        "uid": "test1",
        "fingerprint": "9l3Cy0ROvA7sB1zMhjAWsg==",
        "is_active": true,
        "created_at": "2026-07-14 04:31:49",
        "static_ip": "10.0.0.15",
        "server_ids": ["5ca3580d-8488-457b-85e6-7013239747c4"],
        "rate": {
          "id": "e24f0041-5d00-4f33-8fd4-b58319ce9200",
          "sessions": 12,
          "date_end": "2033-12-12-00:00"
        }
      }
    ],
    "total": 12
  }
  ```

### `GET /user/:id` — Получить профиль конкретного клиента
* **Параметры пути**: `id` — UUID клиента.
* **Ответ (200 OK)**: Возвращает объект клиента (структура аналогична элементу из списка выше).

### `POST /add` — Создать нового клиента
Генерирует пару ключей (Ed25519) и сохраняет слепок в БД. Приватный и публичный ключи шифруются алгоритмом ChaCha20Poly1305 перед записью в СУБД.
* **Тело запроса (JSON)**:
  ```json
  {
    "uid": "Koshka_Vasya",
    "server_ids": ["5ca3580d-8488-457b-85e6-7013239747c4"],
    "rate": null
  }
  ```
* **Ответ (200 OK)**:
  ```json
  {
    "id": "b3254fc0-67bc-4889-a212-054fc3292410",
    "uid": "Koshka_Vasya",
    "fingerprint": "8hJfyOg3vG7n65ww2JiI6+A==",
    "private_key": "T3pMiMHMlo7wsFD0uipDHRbXZB7Jc+zZBrhTxXB3TJ0=",
    "public_key": "enfNE72kYJ8kjyUqlI8HgBZaAnPBmsZVequuoxjZppk=",
    "rate": null
  }
  ```

### `PATCH /user/:id` — Обновить профиль (Бан, Смена имени, Локаций, Static IP)
* **Тело запроса (JSON)** (все поля опциональны):
  ```json
  {
    "uid": "New_Vasya_Name",
    "is_active": false,
    "static_ip": "10.0.0.99",
    "server_ids": ["5ca3580d-8488-457b-85e6-7013239747c4"]
  }
  ```
* **Ответ (200 OK)**: Возвращает обновленный объект клиента.

### `POST /regenerate/:id` — Перевыпустить ключи клиента
Аннулирует старые ключи (клиент больше не сможет по ним подключиться). Генерирует новые и возвращает их.
* **Ответ (200 OK)**: Возвращает объект `RegenerateUserResponse` с новыми ключами и обновленным фингерпринтом.

## 4. Управление тарифами (Rates)

Все запросы требуют заголовка: `Authorization: Bearer <access_token>`.

### `POST /addrate` — Добавить тариф пользователю
* **Параметры запроса (Query)**:
    * `user_id`: UUID клиента.
* **Тело запроса (JSON)**:
  ```json
  {
    "sessions": 5,
    "date_end": "2030-01-01-12:00"
  }
  ```
* **Ответ (200 OK)**: Объект `RateDto`.

### `PATCH /rate/:id` — Обновить существующий тариф
* **Параметры пути**: `id` — UUID тарифа.
* **Тело запроса (JSON)**:
  ```json
  {
    "sessions": 10,
    "date_end": "2035-12-31-23:59"
  }
  ```
* **Ответ (200 OK)**: Объект `RateDto`.

## 5. Публичная раздача (Config Provisioning)

Эти эндпоинты полностью публичны и не требуют авторизации, так как UUID пользователя сам по себе выступает секретным токеном доступа. Если пользователь забанен (`is_active = false`), запросы вернут ошибку `404`.

### `GET /config/:id` — Скачать готовый `client.toml`
* **Параметры пути**: `id` — UUID клиента.
* **Поведение**: Бэкенд на лету расшифровывает приватный ключ клиента из базы данных, загружает базовый шаблон `client_template.toml`, динамически формирует блоки `[[servers]]` для всех привязанных к пользователю нод и отдает результирующий файл в виде вложения (`Content-Disposition: attachment; filename="<uid>.toml"`).

### `GET /config/qr/:id` — Страница сопряжения и QR-кодом
* **Параметры пути**: `id` — UUID клиента.
* **Поведение**: Отдает адаптивную HTML-страницу в OLED-Black дизайне, содержащую кнопку прямого скачивания файла конфигурации и динамически сгенерированный QR-код (ссылающийся на `GET /config/:id`) для быстрого импорта мобильным приложением.

## 6. Межсерверное взаимодействие (Нода <-> Бэкенд)

Эндпоинты используются основным демоном `anet-server` для проверки прав при входящих хендшейках. Запросы защищены токеном ноды.

### `POST /check_access` — Валидация клиента при Handshake
* **Заголовки**: `X-Auth-Key: <AUTH_BACKEND_KEY>`
* **Тело запроса (JSON)**:
  ```json
  {
    "fingerprint": "6l3Cy0ROvA7VB1zMhjAWsg=="
  }
  ```
* **Ответ (200 OK - Допуск разрешен)**:
  ```json
  {
    "allowed": true,
    "message": "OK",
    "static_ip": "10.0.0.15"
  }
  ```
* **Ответ (200 OK - В доступе отказано)**:
  ```json
  {
    "allowed": false,
    "message": "Время действия ключа истекло",
    "static_ip": null
  }
  ```

### `POST /session/start` — Учет начала сессии (+1 устройство)
* **Заголовки**: `X-Auth-Key: <AUTH_BACKEND_KEY>`
* **Тело запроса (JSON)**:
  ```json
  {
    "fingerprint": "6l3Cy0ROvA7VB1zMhjAWsg=="
  }
  ```
* **Поведение**: Увеличивает счетчик одновременных подключений данного пользователя в БД.

### `POST /session/stop` — Учет окончания сессии (-1 устройство)
* **Заголовки**: `X-Auth-Key: <AUTH_BACKEND_KEY>`
* **Тело запроса (JSON)**:
  ```json
  {
    "fingerprint": "6l3Cy0ROvA7VB1zMhjAWsg=="
  }
  ```
* **Поведение**: Уменьшает счетчик одновременных подключений данного пользователя в БД. Применяется каскадно при разрыве или смене транспорта.
